Vediamo le principali regole sintattiche per la definizione di una domanda: >1. Tutto il corpo della domanda deve essere compreso tra parentesi grafe ```{ ... }```; >2. Ogni keyword testuale deve essere compresa tra doppi apici: ```"keywordDiProva"```; >3. L'associazione tra una keyword e il suo valore avviene così: >``` "keywordDiProva" : "valore" ``` >4. Se il valore è numerico allora possono essere omessi i doppi apici: ```"number" : 5```; >5. Per dividere le associazioni è necessario immettere una virgola: ```"key_1" : "valore_1" , "key_2" : "valore_2"; >6. Gli array sono definiti come: valore associato ad una keyword ed elementi che sono compresi tra parentesi quadre ```[]```; >7. Se l'array contiene oggetti, devono essere definiti tra parentesi grafe ```{ }```: >```json [ { oggetto_1 }, { oggetto_2 } ] ``` >8. Se l'array contiene solo ```Stringhe```, devono essere inserite tra doppi apici e separate da virgole: ```json [ "Stringa_1", "Stringa_2" ] ``` ***** ## La semantica Per definire correttamente una domanda è necessario inserire delle parole chiavi che ne definiscono senza ambiguità il contenuto. ```"type" : "tipologiaDomanda"```: questa parola chiave definisce la tipologia di domanda che si vuole realizzare, le tipologie disponibili sono: >- veroFalso; >- rispostaMultipla; >- ordinamentoStringhe; >- ordinamentoImmagini; >- riempimentoSpaziVuoti; >- collegamentoElementi; >- areaCliccabile; >- custom. ***** ## Semantica domanda a risposta vero e falso Per definire una domanda di tipo *veroFalso* è necessario aver inserito le keyword obbligatore. Una volta inserite si può procedere alla stesura del corpo della domanda: >1. Creare un array chiamato ``````answer`````` che conterrà il testo e la soluzione della domanda; >2. All'interno dell'array ```answer``` inserire i seguenti campi obbligatori: > 2.1 ```"text" : "testoDellaDomanda"```: questo definisce il testo in chiaro della domanda che verrà visualizzata; > 2.2 ```"isItRight" : "true|false"```: questo indica se la risposta corretta è vero o falso. Esempio: >```json "answer" : [ { "text" : "testo della domanda", "isItRight" : "risposta esatta" } ] ``` ***** ## Semantica domanda a risposta multipla Per definire una domanda di tipo *rispostaMultipla* è necessario aver inserito le keyword obbligatorie. Una volta inserite si può procedere alla stesura del corpo della domanda: >1. Inserire una keyword ```questionText``` contenente il testo della domanda; >2. Creare una array chiamato ```answer``` che conterrà la lista delle risposte; >3. All'interno dell'array ```answer``` inserire i seguenti campi obbligatori: > 3.1 ```"text" : "testo della risposta"```: questo definisce il testo in chiaro di una possibile risposta; > 3.2 ```"isItRight" : "true|false"```: questo indica se la risposta è corretta. Esempio: >```json "answer" : [ { "text" : "risposta 1", "isItRight" : "true|false" }, { "text" : "risposta 2", "isItRight" : "true|false" } ] ``` la soluzione può contenere più risposte esatte, non c'è limite imposto per questo. ***** ## Semantica domanda a ordinamento di stringhe Per definire una domanda di tipo *ordinamentoStringhe* è necessario aver inserito le keyword obbligatorie. Una volta inserite si può procedere alla stesura del corpo della domanda: >1. Inserire una keyword ```questionText``` contenente il testo della domanda; >2. Creare un array chiamato ```answer``` che conterrà la lista delle parole il loro ordine; >3. All'interno dell'array ```answer``` inserire i seguenti campi obbligatori: > 3.1 ```"text" : "testo della parola"```: questo definisce il testo in chiaro di una parola; > 3.1 ```"position" : n```: questo definiesce la posizione della parola nella stringa. Esempio: >```json "answer" : [ { "text" : "hello", "position" : 1 }, { "text" : "world", "position" : 2 } ] ``` la soluzione può contenere un numero arbitrario di parole da ordinare. ***** ## Semantica domanda a ordinamento di immagini Per definire una domanda di tipo *ordinamentoImmagini* è necessario aver inserito le keyword obbligatorie. Una volta inserite si può procedere alla stesura del corpo della domanda: >1. Inserire una keyword ```questionText``` contenente il testo della domanda; >2. Creare un array chiamato ```answer``` che conterrà la lista delle immagini e la loro posizione; >3. All'interno dell'array ```answer``` inserire i seguenti campi obbligatori: > 3.1 ```"url" : "img/image.png"```: questo definisce l'immagine in chiaro; > 3.2 ```"position" : n```: questo definisce la posizione dell'immagine. Esempio: >```json "answer" : [ { "url" : "img/imm_1.png", "position" : 1 }, { "url" : "img/imm_2.png", "position" : 2 } ] ``` ***** ## Semantica domanda a riempimento spazi vuoti Per definire una domanda di tipo *riempimentoSpaziVuoti* è necessario aver inserito le keyword obbligatorie. Una volta inserite si può procedere alla stesura del corpo della domanda: >1. Inserire una keyword ```questionText``` contenente il testo della domanda; >2. Creare un array chiamato ```answer``` che conterrà gli indici delle parole da oscurare; >3. All'interno dell'array ```answer``` inserire i seguenti campi obbligatori: >3.1 ```"parolaNumero" : n```: questo indica il numero della parola da oscurare nella stringa (l'indice parte da zero). Esempio: >```json "answer" : [ { "parolaNumero" : 0 }, { "parolaNumero" : 5 } ] ``` ***** ## Semantica domanda a collegamento elementi Per definire una domanda di tipo *collegamentoElementi* è necessario aver inserito le keyword obbligatorie. Una volta inserite si può procedere alla stesura del corpo della domanda: >1. Inserire una keyword ```questionText``` contenente il testo della domanda; >2. Creare un array chiamato ```answer``` che conterrà coppie di elementi; >3. All'interno della'array ```answer``` inserire le coppie che dovranno essere collegate: > 3.1 ```"text1" : "testo" , "text2" : "testo"```: questo definisce il collegamento tra 2 stringhe; > 3.2 ```"text1" : "testo" , "url2" : "img/imm_01.png"```: questo definisce il collegamento tra stringa e immagine; > 3.3 ```"url1" : "img/imm_01.png" , "text2" : "testo"```: questo definisce il collegamento tra immagine e stringa; > 3.4 ```"url1" : "img/imm_01.png" , "url2" : "img/imm_02.png"```: questo definisce il collegamento tra 2 immagini. Esempio: >```json "answer" : [ { "text1" : "hello", "text2" : "world" }, { "text1" : "periferica", "url2" : "img/mouse.png" } ] ``` ***** ## Semantica domanda ad area cliccabile Per definire una domanda di tipo *areaCliccabile* è necessario aver inserito le keyword obbligatorie. Una volta inserite si può procedere alla stesura del corpo della domanda: >1. Inserire una keyword ```image``` contenente l'url dell'immagine generale che si vuole rendere cliccabile; >2. Inserire una keyword ```questionText``` contenente il testo della domanda; >3. inserire un array chiamato ```answer``` contenente le coordinate dei punti cliccabili; >4. All'interno dell'array ```answer``` inserire le coppie di coordinate da rendere cliccabili: > 4.1 ```"x" : nx , "y" : ny```: questo definisce un punto nel piano dell'immagine da rendere cliccabile. Esempio: >```json "answer" : [ { "x" : 10, "y" : 20 }, { "x" : 26, "y" : 85 } ] ``` ***** ## Semantica domanda custom Per definire una domanda *custom* è necessario aver inserito le keyword obbligatorie. Una volta inserite si può procedere alla stesura del corpo della domanda. La domanda di tipo *custom* è una domanda che mette in sequenza più tipologie di domande, in modo da rendere più completa una domanda, che per essere risposta correttamente, deve essere corretta in tutte le sue parti. Vediamo come comporre una domanda custom, ricordiamo che all'interno della custom vive la semantica della tipologia di domanda che si intende inserire, perciò non verrà riportato l'esempio, ma solo il riferimento al che spiega la tipologia specifica. >1 Inserire una keyword ```"question"``` che sarà un array di oggetti, dove ogni oggetto sarà una domanda di tipologia nota, vediamo un esempio: >```json "question" : [ { "type" : "veroFalso" }, { "type" : "rispostaMultipla" }, ... ] ``` Possono essere composte anche domande *custom* con ripetizioni di tipologia di domanda interna.
